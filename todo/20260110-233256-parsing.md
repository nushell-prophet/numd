Let's create a command to parse markdown into nushell table.

Try to reuse current helpers as much as possible.

On stdin we get markdown, on output we get table with approximate columns (use proper namings):

- h1-h6
- paragraph
- what else? (anyway it should be standartized with numd/live.nu, though the current live.nu version is just a draft and is a subject to change if necessary)

## User Decisions

1. **Output format**: Block-based (group multi-line content into single rows, like current `parse-markdown-to-blocks`)
2. **Elements to support**: Extended set - h1-h6, paragraph, code blocks, lists (ul/ol), blockquotes
3. **Location**: New file `numd/parse-md.nu`
4. **Frontmatter**: Added as element type with parsed YAML in meta

### Output Table Structure

```
| block_index | element     | content        | meta                         |
|-------------|-------------|----------------|------------------------------|
| 0           | frontmatter | status: draft  | {status: draft, ...}         |
| 1           | h1          | Title          | {}                           |
| 2           | p           | Some text      | {}                           |
| 3           | code        | ls -la         | {lang: nushell, options: []} |
| 4           | ul          |                | {items: [a, b, c]}           |
| 5           | blockquote  | Note text      | {type: note}                 |
```

### Design Choices

1. **Reused patterns from `commands.nu`**:
   - `scan` for stateful transformations (tracking fence/frontmatter state)
   - `window --remainder 2` for detecting block boundaries
   - `group-by block_index --to-table` for grouping lines

2. **State machine approach**: Track `in_fm`, `fm_possible`, `in_code` to handle nested contexts

3. **Frontmatter detection**: Only valid at document start (first line must be `---`)

4. **Meta field contents**:
   - `frontmatter`: parsed YAML record
   - `code`: `{lang, options}`
   - `ul/ol`: `{items}` (ol also has `start`)
   - `blockquote`: `{type}` for GitHub admonitions (`[!NOTE]`, etc.)

5. **Command export**: Uses `export def main` pattern so command is `numd parse-md`

### Usage

```nushell
use numd
open README.md | numd parse-md | table
numd parse-md README.md
```
